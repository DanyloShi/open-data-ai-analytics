# REPORT 3 - Docker and Compose

## Що зроблено

1. Перебудовано проект у сервісну структуру:
- `data_load/`
- `data_quality_analysis/`
- `data_research/`
- `visualization/`
- `web/`
- `common/`

2. Для кожного сервісу додано:
- `app.py`
- `Dockerfile`
- `requirements.txt`

3. Додано [`compose.yaml`](compose.yaml):
- піднімає всі сервіси однією командою;
- використовує Docker network `pipeline_net`;
- використовує volume `pipeline_data` для обміну файлами між контейнерами;
- публікує веб-інтерфейс на `localhost:8080`.

4. Реалізовано мінімальні функції з методички:
- `data_load`: читає CSV, створює таблицю SQLite, імпортує дані;
- `data_quality_analysis`: рахує пропуски/дублікати/некоректні значення;
- `data_research`: робить базову статистику;
- `visualization`: генерує 2 PNG-графіки;
- `web`: показує результати перевірки, дослідження і графіки.

## Як запускати

```bash
docker compose up --build
```

Після старту:
- модулі обробки відпрацюють і завершаться;
- `web` залишиться активним;
- результати доступні в браузері: `http://localhost:8080`.

## Що спростив

- Прибрано зайвий data-science код і складні моделі.
- Основний акцент зроблено на DevOps-механіці: модулі, контейнери, compose, артефакти і відтворюваний запуск.
